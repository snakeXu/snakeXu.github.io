网易微专业-现代前端工程实践方案 解锁webpack-3.1.5-JavaScript的编译,webpack的概念与基础使用

1.编译es6
编译需要安装的loader

npm install babel-loader @babel/core -save-dev

在项目的webpack.config.js文件中配置，现在该文件整体内容如下


module.exports = {
    entry:{
        app:’./app.js’
    },
    outpus:{
        filename:’[name].[hash:8].js’
    },
    module:{
        rules:[
            {
                test:/.js$/,
                use:’babel-loader’
            }
        ]
    }
}

在该文件夹下命令行中输入webpack打包文件，我们打开打包好的文件夹发现，我们写的es6的语法并没有被编译，其原因是我们没有babel-preset。Presets是存储JavaScript不同标准的插件，通过使用正确的presets，告诉babel按照哪个规范编译。presets常见规范:
·es2015
·es2016
·es2017
·env(通常采用)
·babel-preset-stage
通过npm install @babel/preset-env -save-dev安装。同时需要修改配置文件


module.exports = {
    entry:{
        app:’./app.js’
    },
    outpus:{
        filename:’[name].[hash:8].js’
    },
    module:{
        rules:[
            {
                test:/.js$/,
                use:{
                    loader:’babel-loader’,
                    options:{
                        presets:[
                            [@babel/preset-env,{
                                targets:{
                                    browsers:[‘>1%’]
                                }
                            }]
                        ]
                    }
                }
            }
        ]
    }
}

babel-presets的target配置：target是preset的核心配置，告诉preset编译的具体目标。target可以:
·以browsers为目标(通常情况)
·以node版本为目标
·以特定的浏览器为目标
babel-preset只能编译一些语法
2.es6方法的编译
npm install babel-polyfill -save-dev
npm install babel-plugin-transform-runtime babel-runtime –save
babel-polyfill的生效方式:01.生成一个全局对象，该对象定义了一些es6方法的实现，对所有相关方法进行重写。02一般用于项目开发
babel-transform-runtime:01.生成一个局部对象，仅在app.js作用域内生成对象并且只对已经使用的方法进行重写。02一般用于框架开发
另外，实际项目中，由于版本等问题，需要配置的实际依赖往往和文中有区别。如果webpack.config.js的module.rules[0].use.options中内容过多时，关于babel的内容可以在项目根目录中建立.babelrc文件，将babel相关内容配置到这个文件中，webpack.config.js会自动读取该文件内容。
3.编译语法糖
以TypeScript为例，讲解webpack对语法糖的编译
TypeScript是JavaScript的一个超级，让我们可以用强类型的写法写JavaScript，与之类似的有jsx、vue等。
3-1如何编译Typescript
01.安装loader。安装typescript和ts-loader
02.写入配置文件。在webpack.config.js中写入ts-loader
03.编写tsconfig.json。类似于.babelrc,ts-loader的额外配置。
4.一些常见的JavaScript语法糖编译
01.安装对应loader。比如vue-loader/jsx-loader
02.写入配置文件。在webpack.config.js中写入对应loader处理。
03.根据loader规则写配置。可在loader的npm或github上查询
