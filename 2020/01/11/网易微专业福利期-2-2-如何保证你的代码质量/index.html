网易微专业福利期-2.2-如何保证你的代码质量

1.单元测试
·测试是一种验证我们的代码是否可以按预期工作的方法
·单元测试是指对软件中的最小可测试单元进行检查和验证
1-1.前端单元测试的意义
(1)检测出潜在的bug
(2)快速反馈功能输出，验证代码是否达到预期
(3)保证代码重构的安全性
(4)方便协作开发
2.单元测试代码
2-1.被测试对象
被测试对象为程序中最小组成单元，可以是一个函数/方法集/类等


// 被测试对象
let add = (a,b)=>a+b;
// 测试代码
let expect = (res)=>{
    return {
        toBe:(actual)=>{
            if(res !== actual){
                throw new Error(‘期望值与预期值不符’)
            }
        }
    }
}

3.jest的基础使用
3-1.jest的安装
(1)安装node
(2)npm i -D jest或yarn add -D jest进行安装
(3)通过npm ls jest查看是否安装成功
