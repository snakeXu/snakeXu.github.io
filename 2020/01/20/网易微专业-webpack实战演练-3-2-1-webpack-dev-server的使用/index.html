网易微专业-webpack实战演练-3.2.1-webpack-dev-server的使用

1.webpack-dev-server
1-1.什么是webpack-dev-server:项目都要打包上线，所以最好能模拟线上环境进行开发调试。webpack-dev-server就是一个让我们可以模拟线上环境进行项目调试的工具。
1-2.webpack-dev-server提供的常用额外功能
01.路径重定向
02.浏览器中显示编译错误
03.接口代理
04.热更新
1-3.webpack-dev-server使用步骤
01.安装webpack-dev-server
02.配置devServer字段
03.利用命令行开启服务
1-4.devServer常用配置
inline:服务的开启模式
lazy:懒编译
port:代理接口
overlay:错误遮罩(就是黑色浮层上显示错误)
historyApiFallback:路径重定向
proxy:代理请求
hot:热更新
1-5.实战代码
package.json在”scripts”中添加dev命令，命令中–open的作用是运行开发模式的时候，自动帮我们打开浏览器


“scripts”:{
    “test”:”echo \”Error: no test specified\” && exit 1”,
    “dev”:”webpack-dev-server –open”
}

webpack.config.js中添加devServer，与plugins等平级


devServer:{
    port:9001,
    inline:false  //默认为true
}

inline设置为false会有什么不一样呢

我们可以在页面顶部看到一个状态条，显示页面的状态。
如果我们在webpack.config.js中没有设置historyApiFallback,那当我们在浏览器中输入不存在的路由会显示如下页面

当我们设置了


devServer:{
    port:9001,
    historyApiFallback:true
}

当我们在浏览器中输入不存在的路由不会跳转到找不到路径的页面，而是会显示当前页面，或者也可以按照一定正则规则来设置historyApiFallback，比如我们配置的规则是路由如果是/a，那我们跳转到a.html，如果是/b,就跳转到b.html


devServer:{
    port:9001,
    historyApiFallback:{
        rewrites:[
            {
                from:/^\/([ -~]+)/,
                to:function(context){
                    return ‘./‘+context.match[1]+’.html’
                }
            }
        ]
    }
}

3.热更新


devServer{
    hot:true,
    hotOnly:true,//只用热更新
}

extract-text-webpack-plugin与热更新不兼容，把plugins中new extractTextCss({filename:’app.bundle.css’,disable:true})
4.source-map
什么是source-map:为方便调试，我们需要知道打包后的代码对应源文件的位置。如果代码有一处错误，无source-map只能追踪到错误发生在打包后文件的哪个位置，但是打包后的文件不方便阅读。有了source-map就可以查看错误发生在原模块的哪个地方。
配置方法：只需在devServer同级配置devtool即可，devtool的值会有多个选项，下面会有讲解


devtool:’eval-source-map’

4-1.source-map的模式
4-1-1.开发模式
eval
eval-source-map
cheap-eval-source-map
cheap-module-source-map
4-1-2.生产模式
source-map
hidden-source-map
nosource-source-map
