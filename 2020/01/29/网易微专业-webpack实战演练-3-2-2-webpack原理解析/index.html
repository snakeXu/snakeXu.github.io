网易微专业-webpack实战演练-3.2.2-webpack原理解析

1-1.webpack是利用什么来打包的
01.webpack依赖于node的环境与文件操作系统
02.webpack的打包过程，其实就是利用node去读取文件，然后进行一些字符串处理后，再利用node去写入文件
1-2.webpack的打包流程
01.读取文件配置-02.注册内部插件与配置插件-03.loader编译-04.组织模块-05.生成最终文件导出
2.loader原理解析
2-1.loader是如何生效的
loader其实是一个方法:接受一个字符串，方法内部处理完后再返回字符串


module.exports = function(source){
    return source
}

下面我们学习如何写一个loader编译文件,先建一个文件，文件名test.wy，文件内容为


    c(1)

我们要写一个loader编译这个文件。在根文件下建立新文件夹wy-loader。首先在app.js中引入test.wy这个文件，然后打开配置文件webpack.config.js


{
    test:/.wy$/,
    use:’./wy-loader’//相对路径是本地
}

在wy-loader文件夹中建新文件index.js


module.exports = function(resource){
    return resource.replace(‘c’,”console.log”)
}

或者index.js的内容可以为


module.exports = function(resource){
    this.callback(err,resource.replace(‘c’,”console.log”),sourcemap)
    //具体如何写一个loader(有同步loader和异步loader)去详细看webpack的官网
}

3.结果文件分析
3-1.打包代码组成


(function(modules){
    function webpack_require(module){};
    return
        webpack.require(
            webpack_require__.s = “./app.js”
        )
})({
    “./app.js”:(function(){}),
    “mode1.js”:(function(){}),
    “mode2.js”:(function(){}),
})

4.dev-server原理分析
dev-server利用express和一个中间件webpack-dev-middleware来开启服务，然后开启的server会执行打包出来的代码
