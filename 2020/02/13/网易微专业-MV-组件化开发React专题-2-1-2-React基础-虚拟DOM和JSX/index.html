2.虚拟DOM原理
分为三个步骤:
步骤1.把JSX编译成JS
步骤2.把过去构建DOMS浏览器的工作交给JS，由JS构建DOMS
步骤3.把结构数据传入react.render方法
举例1.


< div classname=”cn”>content!< /div >

react将上面的html代码转换成下面的


React.createElement(‘div’,{className:’cn’},’content!’)

举例2.
其实就是比上面的例子代码稍微复杂一些


< div classname=”cn”>content 1!< br >content 2!< /div >


React.createElement(‘div’,{className:’cn’},’content 1!’,React.createElement(‘br’),’content 2!’)

3.Tree Diff算法
3-1.引起DOM树发生变化的三种情况
(1)标签的变化
renderA:< div/ >
renderB:< span/ >
[removeNode < div/ >],[insertNode < span/ >]
(2)组件的变化:不同的组件一般会产生不一样的DOM结构
renderA:< Header/ >
renderB:< Content/ >
[removeNode < Header/ >],[Content < span/ >]
(3)key的优化
old:a,b,c,d
new:b,a,d,c
另外，给元素添加key在很多情况下，可以在性能上得到优化，key元素必须是唯一的不能再key里面赋值一个变量
