2-2.loading组件

这里需要的是组件不用写loading的逻辑，通过继承就能实现相关的功能。首先，我们先踩个坑，我们只写了组件，调用组件的没写，如果需要，自行写一下吧，挺简单的


class LogComponent extends React.Component{
  constructor(props){
    super(props)
    this.state={
      loading:false
    }
  }
  render(){
    const {loading} = this.state
    return (
      < div>
        {loading ? ‘loading…’ : ‘’}
      < /div>
    )
  }
  showloading(){
    this.setState({
      loading:true
    })
  }
  hideloading(){
    this.setState({
      loading:false
    })
  }
}
class App_5 extends LogComponent{
  constructor(props){
    super(props)
  }
  render(){
    return (
      < div>
        app
      < /div>
    )
  }
  componentDidMount(){
    this.showloading()
  }
}
export default App_5;

我们看到，运行之后，页面并没有按照我们预想的显示loading

如果我们把App_5的render()方法注释掉，那么就会显示loading，也就是说，App_5的render方法覆盖掉了LogComponent的render方法。如果我们想同时渲染两个组件，那么就需要做非常简单的修改


class App_5 extends LogComponent{
  constructor(props){
    super(props)
  }
  render(){
    return (
      < div>
        {super.render()} //这里是重点，赶紧划重点
        app
      < /div>
    )
  }
  componentDidMount(){
    this.showloading()
  }
}

这样，我们就得到了想要的效果，如果loading需要隐藏，就简单的加一些代码


class App_5 extends LogComponent{
  constructor(props){
    super(props)
  }
  render(){
    return (
      < div>
        {super.render()} //这里是重点，赶紧划重点
        app
      < /div>
    )
  }
  componentDidMount(){
    this.showloading()
    setTimeout(()=>{
        this.hideloading()
    },3000)
  }
}

3.proxy
看到这里想到，vue2.0版本用的是Object.defineProperty()，今年即将发布的vue3.0记得是换成proxy了，前端er学起来吧，虽然碎碎念的觉得更新如此快的框架之类的，着实有些累人，但是没办法，只能给自己加油鼓劲儿

下面是一个简单的例子，可以直接放在控制台里尝试一下


let obj = new Proxy({
    a:10,
    b:20
}, {
    get: function (target,key,) {
        console.log(‘get ‘,key)
        return target[key] * 10
    },
    set: function (target,key,value) {
        //可以把Reflect理解为Object,在es6中有很多给对象新添加的方法同时部署在了Object和Reflect上，
        //未来会只部署在Reflect上，所以我们要慢慢习惯只用Reflect
        return Reflect.set(target,key,value)
    }
})

4.Decorator

上面代码：我们每次调用Math.add的方法时，都会经过装饰器log的处理。log函数的参数依次为 装饰器装饰的类，类的字段，对象的描述/对象符(包括get/set/valueset/value/hasOwnPrototype等 ) 。我们可以结合代码示例理解为 Math.prototype,add,最后一个老师没说


import React from ‘react’;
import ReactDOM from ‘react-dom’;
let loading = (Component)=>{
  class LoadingComponent extends Component{
    constructor(props){
      super(props)
      this.state={
        loading:false
      }
    }
    render(){
      const {loading} = this.state
      return (
        < div>
          {super.render()} //注意super.render()写在了这里
          {loading ? ‘loading…’ : ‘’}
        < /div>
      )
    }
    showloading(){
      this.setState({
        loading:true
      })
    }
    hideloading(){
      this.setState({
        loading:false
      })
    }
  }
  return LoadingComponent
}
@loading  //装饰器
class App_6 extends React.Component{
  constructor(props){
    super(props)
  }
  render(){
    return (
      < div>
        app
      < /div>
    )
  }
  componentDidMount(){
    this.showloading()
  }
}

let newApp = loading(App_6)
