基本配置文件位于cd /usr/local/nginx/conf/，下面列表的nginx的主配置文件是vim nginx.conf(vim是查看文件内容命令)，下面几个图是关于nginx.conf的介绍
(1)全局区域

在全局区域可以做一些全局的设置，比如#user nobody是一个用户和一个组，user www(一个用户) www(一个组);(这个分号是nginx配置文件后面要加分号结尾)

#使用的用户和组
#user nobody;
#指定工作衍生的进程
worker_processes 1;
#指定错误日志的存放路径
#error_log logs/error.log;
#error_log logs/error.log notice;
#error_log logs/error.log info;
#pid nginx启动起来之后的主进程ID
#pid       logs/nginx.pid;
#指定使用的网络I/O模型
events {
    worker_connections 1024;//允许的最大连接数
}
(2)event区域 (3)http区域(http区域其实是从event区域下面开始到最后，这里截图只有一部分) log_format是用来设置日志的格式，放在http中是全局的，如下图；如果放在http中的server中则表示单个的server的作用 access_log是用来指定日志的格式、存放路径、缓存大小 2-4.gzip压缩 nginx采用的是gzip的压缩，gzip可以压缩到原来大小的30%。进行gzip压缩需要浏览器和服务端都支持。首先由服务端进行压缩，把压缩文件传输到浏览器，浏览器对gizp文件进行解压并且解析，目前为止大部分的浏览器对gzip都是支持的。 关于配置gzip，首先在上面nginx.conf文件中开启gzip on 
gzip on;//开启gzip支持
gzip_min_length 1k;//gzip的最小长度
gzip_buffers 4 16k;
gzip_http_version 1.1;//gzip默认支持的http协议是1.0版本
gzip_comp_level 2;
gzip_types text/plain application/x-javascript text/css application/xml;//设置gzip支持的压缩文件类型
gzip_vary on;
3.Nginx高级应用 3-1.Rewrite重写 首先还是打开conf/nginx.conf文件，找到location 
location / {
    root html;
    rewrite ^(.*).htmp$ /rewrite.html;//把htmp结尾的文件重定向到rewrite.html
    index index.html index.htm;
}
3-2.代理与反向代理 代理就是代理的是客户端；反向代理代理的是服务端 反向代理配置 打开文件 /usr/local/nginx/conf/nginx.conf 
server {
    #端口
    listen 80;
    #主机名
    server_name daili.com;
    location / {
        //http地址也可以写下面upstream中的名字http://myserver，这样相当于一个域名配置多个服务器，即负载均衡
        proxy_pass http://192.168.2.114:8080;
        index index.html index.php;
    }
}
3-3.负载均衡 nginx自带的负载均衡是轮寻的方式。负载均衡就是一个服务器别名可以放多个服务器 配置负载均衡 打开文件 /usr/local/nginx/conf/nginx.conf，在http中配置下面代码 
upstream myserver {
    server 192.168.2.114:8080
    server 192.168.2.114:8081
}
