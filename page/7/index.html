网易微专业-webpack实战演练-3.2.4-代码分割-3.2.5-js代码大小控制

1.一切为了优化
要解决的问题：01.减少加载代码的大小；02.提取公共资源减少加载次数
2.代码分割
2-1.多页面应用
提取公共依赖：把几个页面之中都用到的依赖给打包为一个单独的文件
2-2.单页面应用
减少文件体积，拆分应用：把需要异步加载的内容改成异步加载
2-3.为了业务代码的纯净
有时不希望业务代码里混入第三方代码，或webpack配置代码：把第三方代码和webpack配置代码拆分为单独文件
当我们使用npm run build命令执行之后，发现会打包成三类文件：app.js——业务代码，vendor.js——第三方模块，manifest.js——webpack运行代码
2-4.所以一般这么打包
多页面应用：主业务代码+公共依赖+第三方包+webpack运行代码
单页面应用：主业务代码+异步模块+第三方包+webpack运行代码

4-1.关于tree-shaking(函数式编程更有利于tree-shaking)
moduleb.js


export const a=function(){
    console.log(‘i am a’)
}
export const b=function(){
    console.log(‘i am b’)
}

app.js


import ma from ‘./modulea.js’;
import { a } from ‘./moduleb.js’;
import $ from ‘jquery’;
import _ from ‘loadsh’;
a();

在moduleb.js里我们定义了两个方法，而在app.js中我们只使用了一个方法，那么tree-shaking会帮我们删除掉不用的方法，打包结果里不会包含mb.b，tree-shaking是基于export文档流，所以支持纯函数式(export const a = function(){})这样暴露出去和支持export default{a:function(){},b:function(){}}这样暴露出去。但是如果用面向对象的方式(比如挂载到class.prototype再export出去)这样的方式就不支持。webpack是支持AMD规范的，不支持CMD规范。

网易微专业-webpack实战演练-3.2.3-图片等资源的处理

1.需要用到哪些loader
file-loader:webpack识别除js之外的资源(不用这个loader的时候也是识别js资源的)
url-loader:url-loader包含了file-loader
img-loader:对图片进行压缩处理等需要这个loader
1-1.file-loader
在没有安装这些loader的时候，webpack打包不能正确识别图片资源，示例代码初步了解
webpack.config.js(简单的示例文件，不包含webpack在项目中的完整配置)

module:{
    rules:[
        {
            test:/\.css$/,
            use:extractTextCss.extract({
                fallback:{
                       loader:'style-loader',
                 },
                use:[
                    {
                       loader:'css-loader',
                    },
                ]
            })
        },
        {
            test:/\.(jpg|jpeg|gif|png)$/,
            use:{
                loader:'file-loader'
            }
        }
    ]
}
file-loader的配置项简单示例 
{
    test:/\.(jpg|jpeg|gif|png)$/,
    use:{
        loader:'file-loader',
        options:{
            name:'[name].[hash:4].[ext]',//打包的文件名中包含4位hash值
            outputPath:'assets/img',//打包该资源输出的文件夹
        }
    }
}
常见的坑: (1).路径问题： 
{
    test:/\.(jpg|jpeg|gif|png)$/,
    use:{
        loader:'file-loader',
        options:{
            name:'[name].[hash:4].[ext]',
            outputPath:'assets/img',//打包该资源输出的文件夹
            publicPath:'aaa',//如果定义了该选项，那么引入文件的路径都会以它为路径
        }
    }
}
如果我们按照上面的设置了publicPath，那么打包出来的文件路径就变成了background:url(aaa/img.39e0.jpg)，一般会把outputPath和publicPath设置成一样的 (2).命名问题： 1-2.url-loader 在file-loader的基础上增加了可以将资源转换为base64的功能，默认可以将小于30kb的资源转换为base64 
{
    test:/\.(jpg|jpeg|gif|png)$/,
    use:{
        loader:'url-loader',
        options:{
            name:'[name].[hash:4].[ext]',
            outputPath:'assets/img',
            publicPath:'aaa',
            //我们为什么不把所有东西都转换为base64？
            //虽然我们将资源转换为base64减少了我们的请求，但是资源转换为base64的字符串和资源本身的大小是一样的，
            //比如我们需要的图片是5kb，那么转换为base64之后这5kb会加载到css文件中，css文件变大造成该文件加载变慢
            limit:5000,//控制将多大的资源转换为base64，一般设置为5kb
        }
    }
}
1-3.img-loader 可以压缩图片。loader是从后往前执行的，img-loader只是处理图片，所以我们把img-loader放在url-loader下面。这里要明白img-loader本身不做任何事情，所以并不是写上img-loader就完事儿的，需要配置一系列插件 
{
    test:/\.(jpg|jpeg|gif|png)$/,
    use:[
        {
            loader:'url-loader',
            options:{
                name:'[name].[hash:4].[ext]',
                outputPath:'assets/img',
                publicPath:'aaa',
                limit:5000,
            }
        },
        //压缩图片的插件imagemin系列，对于不同的图片类型有不同的包
        //处理png图片：imagemin-pngquant 处理jpeg图片：imagemin-mozjpeg 处理gif图片：imagemin-gifsicle
        //imagemin系列包都是要通过npm下载安装的
        {
            loader:'img-loader',
            options:{
                plugins:[
                    require('imagemin-pngquant')({
                        speed:2,//控制压缩质量，值越大，对图片压缩程度越小，取值范围为1~11
                    }),
                    require('imagemin-mozjpeg')({
                        quality:2,//控制压缩质量，值越大，对图片压缩程度越小，取值范围为1~100
                    }),
                    require('imagemin-gifsicle')({
                        optimizationLevel:2,//取值范围为1,2,3，推荐设置为1，设置为3极有可能失真
                    }),
                ]
            }
        }
    ]
}

const routerLock = {
    beforeEach(to, from, next) {
        const matched = to.matched || [];
        const locks = [];
        matched.forEach((item) => {
            if (item.meta && item.meta.locks) {
                const itemLocks = item.meta.locks;
                itemLocks.forEach((lock) => {
                    if (lock.params && lock.params.length) {
                        locks.push(lock);
                    }
                });
            }
        });
        if (!locks.length) {
            next();
            return;
        }
        const includePath = function (path) {
            if (typeof path === ‘string’) {
                return path === to.name || path === to.path;
            }
            if (path instanceof RegExp) {
                return path.test(to.name) || path.test(to.path);
            }
            return false;
        };
        const fromQuery = from.query;
        const toQuery = to.query;
        let isChanged = false;
        locks.forEach((lock) => {
            // 没有 include 时全通过
            if (lock.include && !lock.include.some(includePath))
                return;
            if (lock.exclude && lock.exclude.some(includePath))
                return;
            lock.params.forEach((param) => {
                if (!(param in toQuery)) {
                    isChanged = true;
                    toQuery[param] = fromQuery[param];
                }
                if (toQuery[param] === ‘’) {
                    toQuery[param] = undefined;
                } // 美化路由
            });
        });
        if (isChanged) {
            next({
                path: to.path,
                query: toQuery,
            });
        } else {
            next();
        }
    },
    update(query, route, router) {
        if (Object.keys(query).some((key) => query[key] !== route.query[key])) {
            router.push({
                path: route.path,
                query: {
                    …route.query,
                    …query,
                },
            });
        }
    },
    install(Vue) {
        Vue.prototype.$routerLock = function (query) {
            return routerLock.update(query, this.$route, this.$router);
        };
    },
};
export default routerLock;

< div :class=”$style.list” ref=”list” @scroll=”onScroll”>
    < div :class=”$style.scroll” :style=”{ paddingTop: paddingTop + ‘px’, paddingBottom: paddingBottom + ‘px’ }”>
        < div :class=”$style.item” v-for=”item in viewItems” :key=”item.id” :even=”!!(item.id%2)”>

        < /div>

< script>
export default {
    props: {
        list: Array,
        viewCount: { type: Number, default: 100 },
    },
    data() {
        return {
            minHeight: 20,
            viewIndex: 0,
            paddingTop: 0,
            paddingBottom: 0,
        }
    },
    computed: {
        viewItems() {
            return this.list.slice(this.viewIndex, this.viewIndex + this.viewCount);
        },
    },
    methods: {
        onScroll(e) {
            const listEl = e.target;
            // 记录当前 DOM 节点的高度
            const children = Array.from(listEl.children[0].children);
            children.forEach((childVM, index) => {
                if (this.list[this.viewIndex + index])
                    this.list[this.viewIndex + index].height = childVM.offsetHeight;
            });
            / eslint-disable /
            const scrollTop = listEl.scrollTop;
            let accHeight = 0;
            let viewIndex = this.viewIndex;
            for (let i = 0; i < this.list.length; i++) {
                const item = this.list[i];
                accHeight += item.height || this.minHeight;
                if (accHeight > scrollTop) {
                    viewIndex = i;
                    break;
                }
            }
            if (viewIndex - this.viewIndex >= 0 && viewIndex - this.viewIndex < this.viewCount / 2)
                return;
            let paddingTop = 0;
            let paddingBottom = 0;
            for (let i = 0; i < this.list.length; i++) {
                const item = this.list[i];
                if (i < viewIndex) {
                    paddingTop += item.height || this.minHeight;
                } else if (i >= viewIndex + this.viewCount) {
                    paddingBottom += item.height || this.minHeight;
                }
            }
            this.viewIndex = viewIndex;
            this.paddingTop = paddingTop;
            this.paddingBottom = paddingBottom;
        },
    },
}
< /script>
